<task type="auto">
  <name>Task 2: Fix PTT daemon path in main.js for production builds</name>
  <files>electron-client/main.js</files>
  <action>
In electron-client/main.js, the `tryRustDaemon()` method currently hardcodes the daemon path to `target/debug`:

```javascript
const daemonPath = path.join(__dirname, 'ptt-daemon', 'target', 'debug', 'ptt-daemon');
```

Replace this single line with a path that correctly locates the binary in both dev (target/debug) and packaged (target/release via asar.unpacked) modes:

```javascript
const daemonPath = app.isPackaged
  ? path.join(process.resourcesPath, 'app.asar.unpacked', 'ptt-daemon', 'target', 'release', 'ptt-daemon')
  : path.join(__dirname, 'ptt-daemon', 'target', 'debug', 'ptt-daemon');
```

This is line 58 in the current file, inside the `tryRustDaemon()` method. Replace only that one `const daemonPath = ...` line. Leave all surrounding code (the `if (process.platform !== 'linux')` guard, the `fs.existsSync` check, etc.) unchanged.

Rationale:
- `app.isPackaged` is Electron's built-in flag for detecting packaged vs dev mode
- When packaged, electron-builder places asarUnpack files in `resources/app.asar.unpacked/`
- `process.resourcesPath` resolves to the `resources/` directory inside the app bundle
- Dev mode: daemon is built with `cargo build` (debug profile) for rapid iteration
- CI: daemon is built with `cargo build --release`, placed in target/release/

Create a minimal valid 512x512 PNG icon. Use Node.js to generate a minimal valid PNG file programmatically:

```bash
mkdir -p electron-client/build

node -e "
// Minimal valid 1x1 blue PNG, scaled to 512x512 via IHDR
// Using a real minimal PNG structure
const { execSync } = require('child_process');

// Create a simple SVG and convert, or write a minimal binary PNG
// Approach: write a valid minimal PNG header for a 1x1 solid blue pixel
const PNG_1x1_BLUE = Buffer.from([
  0x89, 0x50, 0x4E, 0x47, 0x0D, 0x0A, 0x1A, 0x0A, // PNG signature
  0x00, 0x00, 0x00, 0x0D, 0x49, 0x48, 0x44, 0x52, // IHDR chunk length + type
  0x00, 0x00, 0x02, 0x00, // width: 512
  0x00, 0x00, 0x02, 0x00, // height: 512
  0x08, 0x02,             // bit depth: 8, color type: RGB
  0x00, 0x00, 0x00,       // compression, filter, interlace
  0x00, 0x00, 0x00, 0x00, // placeholder CRC (electron-builder may accept this)
], 'hex');
"
```

The above approach is complex. Instead, use a simpler method: create a minimal valid PNG using base64-encoded binary data. The following is a known-good minimal 32x32 solid blue PNG in base64:

```bash

// Minimal valid 32x32 solid blue PNG (base64)
```
