```yaml
# .github/workflows/release-docker.yml
# Builds the Phoenix server Docker image and pushes to GitHub Container Registry.
# Triggered on semver release tags (e.g., v1.2.0).
#
# After first push, make the package public:
# GitHub → Repository → Packages → cromulent → Package settings → Change visibility → Public
name: Release Docker

on:
  push:
    tags:
      - 'v*.*.*'

jobs:
  docker:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write

    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Log in to GitHub Container Registry
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Extract Docker metadata
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ghcr.io/${{ github.repository }}
          tags: |
            type=semver,pattern={{version}}
            type=semver,pattern={{major}}.{{minor}}
            type=raw,value=latest

      - name: Build and push Docker image
        uses: docker/build-push-action@v6
        with:
          context: .
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
```

Key design decisions:
- `on.push.tags: ['v*.*.*']` — only fires on semver release tags, not every push
- `permissions: packages: write` — required for GITHUB_TOKEN to push to GHCR; `contents: read` for checkout
- `docker/login-action@v3` — official Docker action; uses `ghcr.io` registry with automatic GITHUB_TOKEN
- `docker/metadata-action@v5` — produces three tags: exact version (v1.2.0 → 1.2.0), major.minor (1.2), and `latest`
- `docker/build-push-action@v6` — builds from repo root (`context: .`) using the existing Dockerfile
- No external PAT secrets needed — GITHUB_TOKEN with `packages: write` is sufficient for GHCR

The comment at the top of the file reminds the operator to make the package public after first push (GHCR defaults to private — this is a one-time manual step that cannot be automated without a PAT).

Create the `.github/workflows/` directory structure as needed. The file must be valid YAML — verify indentation carefully.
