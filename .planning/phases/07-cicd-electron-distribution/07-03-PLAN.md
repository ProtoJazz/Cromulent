```yaml
# .github/workflows/release-electron.yml
# Builds Electron desktop client for Linux (.AppImage, .deb) and Windows (.exe NSIS).
# Publishes artifacts to a GitHub Release automatically.
# Triggered on semver release tags (e.g., v1.2.0).
#
# Prerequisites for runners:
# - Linux: ubuntu-latest (Rust daemon compiled via cargo; uiohook-napi rebuilt by electron-builder)
# - Windows: windows-latest (uiohook-napi rebuilt by electron-builder; VS Build Tools pre-installed)
#
# No code signing configured for v1.1. Windows users will see SmartScreen "Unknown Publisher"
# warning on first run — this is expected for unsigned open-source builds.
name: Release Electron

on:
  push:
    tags:
      - 'v*.*.*'

jobs:
  build:
    strategy:
      fail-fast: false
      matrix:
        include:
          - os: ubuntu-latest
            platform: linux
          - os: windows-latest
            platform: win
    runs-on: ${{ matrix.os }}
    permissions:
      contents: write

    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'
          cache-dependency-path: electron-client/package-lock.json

      # Linux only: compile Rust PTT daemon (evdev is Linux-only; not needed on Windows)
      - name: Install Rust toolchain
        if: matrix.platform == 'linux'
        uses: dtolnay/rust-toolchain@stable

      - name: Build PTT daemon (Linux only)
        if: matrix.platform == 'linux'
        working-directory: electron-client/ptt-daemon
        run: cargo build --release

      # Install Node dependencies and build+publish Electron installers
      # electron-builder auto-detects OS and builds appropriate targets:
      #   Linux  → AppImage + deb (from package.json build.linux.target)
      #   Windows → nsis (from package.json build.win.target)
      # --publish always: create GitHub Release and upload artifacts immediately
      # GH_TOKEN: GITHUB_TOKEN has contents:write permission (set above)
      - name: Install dependencies and publish
        working-directory: electron-client
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          npm ci
          npx electron-builder --publish always
```

Key design decisions:
- `strategy.fail-fast: false` — Linux build failure should not cancel Windows build and vice versa; both are independent platform deliverables
- `permissions: contents: write` — required for electron-builder to create GitHub Releases and upload assets via GITHUB_TOKEN
- `actions/setup-node@v4` with `cache: 'npm'` and `cache-dependency-path: electron-client/package-lock.json` — caches node_modules by platform
- `dtolnay/rust-toolchain@stable` — use `dtolnay` (actively maintained) NOT `actions-rs/toolchain` (archived)
- `if: matrix.platform == 'linux'` on both Rust steps — evdev crate does not build on Windows; Windows PTT uses uiohook-napi
- `cargo build --release` in `electron-client/ptt-daemon` — binary lands at `electron-client/ptt-daemon/target/release/ptt-daemon` where electron-builder's files config will pick it up
- `npm ci` — clean install from lock file, ensures reproducible builds
- `npx electron-builder --publish always` — electron-builder reads package.json build config, compiles for the current OS, then uploads to the GitHub Release referenced by the tag; `releaseType: "release"` in package.json (set by Plan 01) prevents draft creation
- No `--linux` or `--win` CLI flag — electron-builder detects the OS automatically from the runner; specifying mismatched targets would cause errors

The comment about SmartScreen warning is intentional — informs future maintainers that code signing is deferred to a future milestone.
